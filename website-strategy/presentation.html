The Proposal

One Hub.
Infinite Worlds.

"Simple to use. Simple to manage. Sophisticated in possibilities."

A unified hub at bohemia.net doesn't mean every game looks the same. It means every game benefits from the same solid foundation — shared navigation, shared accounts, shared analytics — while retaining complete visual independence.

The key distinction is between two types of content under the hub:

Web Presentations

Marketing-focused pages with brand identity, news, trailers, game profiles. Lives at bohemia.net/games/dayz

vs.
Web Applications

Standalone tools with own backend — Workshop, Feedback Tracker, Account system. Can deviate in design & infrastructure.

Format 01

Presentation
Website

Shared Navigation Full Brand Identity News & Blog Permanent Own Backend

The core game presence within bohemia.net. Each game gets its own fully branded section — distinct color palette, typography, visual language — while sharing the hub's navigation skeleton, account system, and CMS backend.

Think ioi.dk: HITMAN and 007 First Light have completely different looks, but the same parent nav, same account login, same news infrastructure. Or rockstargames.com: GTA VI and RDR2 coexist with bold individual identities under one domain.

Format 02

Landing
Page

Shared Navigation Maximum Freedom Dynamic Content Campaign-driven Short-lived

A single-purpose, conversion-focused page for a specific moment: a game announcement, a DLC launch, a pre-order campaign. Navigation is minimal or absent — the goal is one action.

The page lives under the hub domain but behaves autonomously. Brand managers get maximum creative freedom — custom animations, unique layouts, full-bleed media — with no design constraints from the hub system.

Format 03

Microsite

Shared Navigation Maximum Freedom Multi-page Event / Campaign Temporary

A small, self-contained web experience — typically 3–7 pages — built for a specific moment in a game's lifecycle. Events, DLC launches, anniversary celebrations, esports tournaments.

Unlike a landing page, a microsite has its own light navigation and can include interactive elements, community features, or special tools. It lives under the hub domain but is fully visually decoupled.

Format 04

Web
Application

Own Backend Real-time User Auth Hub CMS Standalone

Functional tools that require their own server, database, and backend logic. These are not marketing pages — they're software that runs in the browser. Design can deviate completely from the hub.

These will always exist as their own units within the ecosystem. The hub links to them; they optionally share the Bohemia Account system for login.
