Generally the hub is a combination of web presentations (marketing websites for each of our game brands). Web presentations have static marketing content (= project presentation, landing pages or microsites for campaigns etc) and dynamic content (blogs, patch notes, community features etc). Web presentations should fit within a unified systemic design & use roughly the same feature set, but allow for creativity in terms of content presenation. Good example are the websites of ioi.dk and https://www.rockstargames.com/. They are simple in layout, navigation, but offer space for web presentation creativity in a unified enviroment.

Alongside web presentations, we will also have web applications. These are a bit more standalone, they will run on heir own server, own backend, can deviate in brand design. Typical example is the Arma Reforger Modding Workshop https://reforger.armaplatform.com/workshop

My goal of the demo we should develop quickly today is to demonstrate the advantages of this approach, and assure brand managers that they are not losing anything in terms of creativity of their web presentations.

We have some designs in figma we can work with, but I'd just like to build a simple demo prototype from scratch possibly, using actual live content about our games.
